# CNN Heatmap People Counter

Spatial CNN with **temporal information via frame stacking** for radar heatmap classification. This architecture processes multi-frame aggregated heatmaps (8 frames) to capture motion patterns without explicit temporal layers (LSTM/TCN), combining temporal context with lightweight spatial processing using GlobalAveragePooling.

## Architecture

**Model Type:** Spatial CNN with Frame-Stacked Temporal Context

### Network Structure

```
Input: 64×64×1 Heatmap (8 frames aggregated)
    ↓
[Conv2D(12) + BN + MaxPool]
    ↓
[Conv2D(24) + BN + MaxPool]
    ↓
[Conv2D(48) + BN]
    ↓
GlobalAveragePooling2D  ← Key: Prevents spatial overfitting
    ↓
[Dense(64) + Dropout(0.4) + L2]
    ↓
Softmax(3 classes)
```

**Key Features:**
- **Frame Stacking**: 8 consecutive frames aggregated into single heatmap (temporal context embedded)
- **GlobalAveragePooling**: Replaces Flatten to reduce spatial sensitivity
- **L2 Regularization**: Prevents weight explosion (λ=0.001)
- **Class Weighting**: Handles imbalanced people count distribution
- **Custom Macro-F1**: True multi-class F1, not accuracy
- **Log Preprocessing**: Enhances sparse radar signal contrast

**Parameters:** ~50K (extremely lightweight compared to LSTM/TCN)

## Temporal Processing Method

Unlike CNN-LSTM/TCN which use explicit temporal layers, this model embeds temporal information through **frame aggregation** during dataset generation:

### Frame Stacking Pipeline

```
[Frame t-7] ──┐
[Frame t-6] ──┤
[Frame t-5] ──┤
[Frame t-4] ──┼──> Aggregation ──> Single 64×64 Heatmap ──> CNN
[Frame t-3] ──┤    (concat/mean/
[Frame t-2] ──┤     sum/max)
[Frame t-1] ──┤
[Frame t  ] ──┘
```

**Advantages:**
- Temporal patterns captured without recurrent layers
- Single-pass inference (no sequence processing overhead)
- Consistent input size regardless of frame count
- Compatible with standard CNN architectures

### Generate Your Own Dataset

Configure frame aggregation in heatmap dataset generator:

```python
# Example: 8-frame concatenation mode
config = HeatmapConfig(
    input_dir="parsed_data",
    output_dir="heatmap_64x64_8frames",
    resolution=64,                    # Image resolution
    image_x_limits=(-6.0, 6.0),      # Spatial range X (meters)
    image_z_limits=(-6.0, 6.0),      # Spatial range Z (meters)
    intensity_feature="snr",          # or "velocity"
    aggregate_frames=True,            # Enable frame stacking
    frames_per_sample=8,              # Number of frames to aggregate
    frame_stride=1,                   # Stride between samples
    aggregation_mode="sum"         # concat/mean/sum/max
)
```

**Aggregation Modes:**
- `concat`: Concatenate point clouds before heatmap generation
- `mean`: Average heatmap intensities across frames
- `sum`: Sum heatmap intensities across frames (recommended)
- `max`: Maximum intensity projection across frames

See [heatmap_dataset_generator/README.md](../heatmap_dataset_generator/README.md) for detailed configuration.

## Model Architecture Details

### Why GlobalAveragePooling?

**Problem:** Standard CNNs with `Flatten()` overfit to **spatial noise patterns** in radar heatmaps (multipath, clutter).

**Solution:** `GlobalAveragePooling2D` averages each feature map into a single value, making the model learn "presence of features" rather than "location of features."

**Result:** Better generalization to unseen environments and sensor positions.

### Preprocessing Pipeline

```python
# 1. Log transform (enhances weak radar returns)
features = np.log1p(features)

# 2. Normalize by training max (prevents data leakage)
features = features / train_max
```

**Critical:** Normalization uses **only training statistics** to avoid test set leakage.

### Inference

```python
import numpy as np
from tensorflow.keras.models import load_model

# Custom metric required for loading
class MacroF1(tf.keras.metrics.Metric):
    # ... (copy from training script)

model = load_model('best_cnn_heatmap_model.h5', 
                   custom_objects={'MacroF1': MacroF1})

# Load and preprocess new heatmap
heatmap = np.load('new_heatmap.npy')  # Shape: (64, 64, 1)
heatmap = np.log1p(heatmap)
heatmap = heatmap / train_max  # Use same normalization as training
heatmap = np.expand_dims(heatmap, axis=0)  # Add batch dimension

# Predict
pred = model.predict(heatmap)
people_count = np.argmax(pred) + 1  # Convert 0,1,2 → 1,2,3
print(f"Detected {people_count} people")
```

## Performance Characteristics

### Strengths

- **Temporal context without recurrent layers**: Frame stacking embeds motion patterns
- **Fast inference**: ~5ms per frame on GPU (single-pass, no sequence processing)
- **Low memory footprint**: ~50K parameters vs 180-220K for LSTM/TCN
- **Robust to spatial noise**: GlobalAveragePooling prevents location overfitting
- **Flexible temporal window**: Easy to adjust frame count in dataset generator

### Limitations

- **Fixed temporal window**: 8 frames baked into heatmap (cannot adapt dynamically)
- **No explicit motion modeling**: LSTM/TCN learn temporal transitions explicitly
- **Aggregation information loss**: Some temporal details lost in frame merging
- **Requires retraining for different frame counts**: Cannot change temporal window at inference
