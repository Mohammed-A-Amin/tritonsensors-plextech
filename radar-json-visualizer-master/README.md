**TI Radar Data Parser & Visualizer**

This is a Python-based interactive web application built with Streamlit to parse, analyze, and visualize data captured from a Texas Instruments radar sensor, specifically in a ceiling-mounted configuration (tested on **IWR6843AOPEVM ceiling-mounted** configuration).

This tool replays data captured from a JSON file generated by the TI Out-of-Box (OOB) demo. It allows users to scrub through frames and examine different outputs from the radar's signal processing chain.

**Features**

* Interactive Frame Selection: Use a slider to navigate through all the frames in the dataset.

* Point Cloud Visualization: View the detected points in 3D, top-down (X-Z), and side-on (X-Y) plots.

* Dynamic Point Cloud Coloring: Color points based on their velocity or intensity to add another dimension to the analysis.

* Processed Data Plots:

  * Range & Noise Profile: Inspect the signal and noise across the detection range.

  * Range-Doppler Heatmap: Visualize the velocity of objects at different ranges.

  * Range-Azimuth Heatmap: See the angular position of objects in the horizontal plane.

  * Azimuth-Elevation Heatmap: Understand the top-down angular distribution of signals.

* Overall Statistics: View a summary plot of detected points per frame and key statistical metrics (mean, median, min, max, std dev).

* Configurable Display: Toggle between linear magnitude and dB scale for heatmap visualizations.

**Input Data Format**

The application requires a single JSON file containing the multi-frame radar data. The JSON file must have a top-level key named "data", which holds a list of frame objects.

Each frame object within the list should have a structure similar to this:
```
{
  "frameData": {
    "frameNum": 1,
    "numDetectedPoints": 5,
    "pointCloud": [
      [x1, y1, z1, velocity1, intensity1],
      [x2, y2, z2, velocity2, intensity2],
      ...
    ],
    "rangeProfile": [...],
    "noiseProfile": [...],
    "rangeDopplerHeatmap": [...],
    "azimuthElevationStaticHeatmap": [...]
  }
}
```

* ```pointCloud```: Each inner list represents a detected point with its x, y, z coordinates, velocity, and intensity.

* ```rangeProfile```, ```noiseProfile```, etc.: These are flattened arrays of the corresponding heatmap/profile data.

An example data file, ```replay_1_randomwalk.json```, is expected by default.

**Output**

The script launches a web-based GUI in your browser. The output is a series of interactive plots and data tables, as described in the "Features" section. There are no file outputs; the results are purely for visualization and analysis within the app.

**Requirements**

The script is written in Python and requires the following libraries:

* streamlit

* pandas

* numpy

* plotly


**Setup and How to Run**

1- Install Dependencies:

If you don't have the required libraries, install them using pip:
```
pip install streamlit pandas numpy plotly
```
2- Place Data File:

Ensure your radar data JSON file (e.g., ```replay_1_randomwalk.json```) is in the same directory as the Python script (```ti_radar_visualizer.py```).

3- Run the Application:

Open a terminal or command prompt, navigate to the directory containing the script, and run the following command:
```
streamlit run ti_radar_visualizer.py
```

4- View in Browser:

Streamlit will automatically open a new tab in your default web browser with the running application. If it doesn't, the terminal will provide a local URL (usually http://localhost:8501) that you can visit.
